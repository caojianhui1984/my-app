# 基于 rPPG 的 Mac 摄像头网页端心率测量

这是一个纯前端 rPPG（remote Photoplethysmography，远程光电容积脉搏波）示例应用，可通过网页端直接调用 Mac 前置摄像头采集用户面部 RGB 视频，并在浏览器本地实时估计心率。

> 目标：在光照均匀、面部基本稳定、摄像头 24–30fps 的条件下，心率实时采集最大误差目标为 ±5 bpm。实际精度受摄像头、浏览器、环境光、运动幅度和个体差异影响；本项目不能替代医疗设备。

## 功能特性

- **WebRTC 摄像头采集**：使用 `navigator.mediaDevices.getUserMedia` 直接请求 Mac 前置摄像头权限。
- **面部 ROI 自动定位**：优先使用浏览器 `FaceDetector` API，若不可用则回退到中心人脸先验区域。
- **鲁棒肤色 ROI 筛选**：将额头/面颊区域切成网格，过滤低亮度、过曝和非肤色块，降低背景、头发、眼镜和阴影干扰。
- **环境光抑制**：对 RGB 信号做归一化和移动平均去趋势，减少慢变光照影响。
- **运动鲁棒心率估计**：采用 POS（Plane-Orthogonal-to-Skin）色彩投影生成脉搏信号，并通过 45–180 bpm 频带搜索估计主频。
- **实时质量反馈**：显示信号质量、采样时长、有效帧率、ROI 策略和心率置信度，帮助用户调整姿态和光照。
- **隐私友好**：所有视频帧都只在本地浏览器内处理，不上传服务器。

## 快速开始

由于浏览器只允许在 HTTPS 或 `localhost` 环境下调用摄像头，请使用本地静态服务器启动：

```bash
python3 -m http.server 8080
```

然后在 Mac 的 Chrome、Edge 或 Safari 中打开：

```text
http://localhost:8080
```

点击 **开始测量**，授权摄像头后保持面部稳定约 10–30 秒，即可看到心率读数。

## 算法流程

1. **视频采集**：请求 1280×720、24–30fps 的前置摄像头视频流。
2. **人脸/ROI 定位**：使用 FaceDetector 获取人脸框；若浏览器不支持，则使用画面中央的人脸先验框。
3. **肤色网格采样**：在上半脸区域提取 RGB 网格均值，按肤色规则、亮度范围和色彩分布筛选高质量网格。
4. **时间序列构建**：保存最近 30 秒 RGB 均值，自动剔除过旧样本。
5. **归一化与 POS 投影**：对 RGB 通道去均值归一化，使用 POS 投影增强脉搏相关分量并抑制共同光照变化。
6. **去趋势与频域估计**：移动平均去趋势后，在 45–180 bpm 频带内搜索最大功率主频。
7. **置信度与平滑**：依据峰值能量占比和平均 ROI 质量计算置信度，对输出心率做指数平滑，减少跳变。

## 提高 ±5 bpm 目标达成率的建议

- 使用稳定的自然光或无频闪补光，避免强背光。
- 摄像头保持 24fps 以上，测量窗口至少 10 秒，推荐 20–30 秒。
- 用户脸部占画面高度 35%–60%，额头和双颊尽量无遮挡。
- 测量期间减少大幅转头、说话、表情变化和手遮挡。
- 若信号质量显示“偏弱”，请改善光照、靠近摄像头或重新开始采集。

## 文件结构

```text
.
├── index.html      # 页面结构和摄像头/读数 UI
├── styles.css      # 响应式深色视觉样式
└── src/rppg.js     # rPPG 采集、ROI、POS、频域估计和实时渲染逻辑
```

## 部署到 GitHub Pages

1. 将仓库推送到 GitHub。
2. 在仓库 Settings → Pages 中选择当前分支和根目录。
3. 等待 GitHub Pages 构建完成后，通过 HTTPS 地址访问页面。

## 注意事项

- rPPG 对环境和动作敏感，本项目通过 ROI 筛选、POS 投影、去趋势、频带约束和质量门控提升鲁棒性，但无法保证所有场景都达到 ±5 bpm。
- Safari 对 `FaceDetector` 支持可能不同；不支持时应用会自动使用中心人脸先验 ROI。
- 若部署到非 localhost 域名，必须使用 HTTPS，否则浏览器会拒绝摄像头权限。
